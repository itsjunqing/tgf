Focus Question 2

Is it ever worth it to manage your own private cloud services?

There are two types of cloud system:

  1. Private Cloud: In a private cloud (or known as an internal or enterprise cloud), the data resides on the company's intranet or hosted data center where all of the data is protected behind a firewall.
  2. Public Cloud: In a public cloud, the data resides in the provider’s data center and the provider is responsible for the management and maintenance of the data center. For instance, services such as Dropbox, Google Drive, iCloud, etc.

The biggest trade-off of not using public cloud is the cost associated with setting up a private cloud. Since the providers of public cloud offer a pay as you go solution, companies can save on the huge upfront cost for the IT infrastructure needed to deploy their own cloud services and do not need cloud engineers to maintain it.

Public cloud can add more storage when needed at a very minimal cost, because the cloud providers specialize in this service and they have a huge user base which they are able to set up the servers for storage expansion. It is possible that storage space is under utilized in a private cloud since it is fixed at the time of setup and difficult to expand so the public cloud is scalable according to the demand of the user.

Since the public cloud is hosted over the internet it can be accessed by anyone around the globe which makes it easier for the company to make their content available to customers worldwide at a low cost. Using a private cloud would need a dedicated data centre inside the company’s premises so it would take up a lot of physical space.

Public cloud is valuable because it is constantly updated, which makes the services and capabilities of public cloud constantly up to date from a tech, accessibility and governance perspective. This also allows public cloud users to potentially integrate with platform-as-a-service capabilities such as database, machine learning and other microservices that are native to the public cloud more easily.

Private cloud gives you more control over your data since you are managing it yourself, the storage which you get is hosted from a third party but the infrastructure on top is deployed by the company so you manage the cloud yourself. It requires more resources and a bigger team but in this way the data is more secure when compared with the public cloud since all the managing is done by the public cloud provider and you can not set custom policies for access like you would in a private cloud.

The private cloud that we have setup is ownCloud. ownCloud is an open source cloud provider where the source code for the platform is visible toe everyone. This makes it easy for programmers around the world to see and pinpoint any bugs allowing the ownCloud engineers to fix them accordingly. This private cloud - ownCloud has a strong focus on security and data privacy. Most of the large corporations - cloud service providers like Google Drive, Dropbox, etc are hosted in the United States and there do not exist strong privacy laws which protect the data of the user.

In contrast to most cloud providers — who aggregate data from different customers to minimize storage needs — ownCloud is single tenant based and provides separate cloud instances with dedicated private storage for each customer. An Owncloud instance is a dedicated installation with dedicated storage and user space. It is only accessible by the user. Other Owncloud users have their own dedicated workspace so no cross-cloud access is possible.

Such benefits over privacy and security comes with a cost that we have seen during the installation process because it may take certain people some time to setup a private cloud from scratch. On top of that, maintainence cost is an issue when hosting a private cloud too.

In conclusion, the choice between public cloud and private cloud mainly depends on the needs of the organization. If it is a small company which still has time to grow then using a public cloud would be the more cost-effective and flexible option whereas for large organizations who value privacy and data integrity the hassles of private cloud seem to be justified.
