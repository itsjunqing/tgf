Ethical Implications (with Ethical Reasoning Framework)

What are the ethical implications when using a self hosted cloud?

Ethical Reasoning Framework is utilized here to critically evaluate the ethical implications resulted from building a self-hosted cloud storage. It comprises of a cycle of five steps:

  1. Problem Defintion: This section critically defines the ethical issue that occur in reference to the code of ethics
  2. Gathering Information: This section gathers all the required information that argues on the ethical issues
  3. Applying Guiding Principles: This section talks about the importance of moral virtues and how to further improve upon the problem, evaluating the basics of ethics to make a good and right decision
  4. Generating, Evaluating and Taking Action: This section talks about the possible viable solutions that could potentially solve the issue arised from hosting a private cloud
  5. Reflection: This section reflects upon the overall process of this framework with analysis on our decided solution with consideration of any additional problems which might still occur

Problem Defintion

The central problem of utilizing ownCloud as our cloud storage solution is that ownCloud is an open-source. Open source means that its source code is available online and this could create a risk for ethical problems to arise. As the source code is accessible at the developers’ fingertips, it potentially allowed a situation where the developers recognizing the bugs residing in the source code but not reporting it to the maintainers of the source code, failing to comply with ACM Code of Ethics (1.3: Be honest and trustworthy).

Such behavior may be deemed to be risky for consumers whose data is in being stored under an open-source platform because the developers that successfully identified the bugs may exploit it to conduct unethical activities such as breaching online users’ private data, violating (1.6 Respect privacy), resulting the data to be insecure. As the core purpose of building private cloud storage is to reinforce the security aspects of managing data, such possible exploitation contradicts the intention of a private cloud.

Gathering Information

The issue of a breach happening due to an overlooked bug in an open-source is not uncommon. Developers generally overlook the importance of bugs in the source code and did not invest sufficient time to maintain and update them. Often organizations also fail to recognize and acknowledge that open source exploitations are vulnerable in certain ways when incorporating these platforms into their applications. Although the number of people involved in the open-source community to pinpoint the vulnerabilities is huge, if the updates are not frequent, those critical bugs are exposed to hacking, allowing hackers to gain private information from attempting to breach services like this.

This fact is proven especially by the more notable news regarding the Equifax data breach. Equifax Inc. is one of the three largest consumer credit reporting agencies and is an open-source. However, in March 2017, Equifax was hit with a huge data breach with more than hundreds of millions of personal information being stolen. The source of the issue can be led down to Equifax failing to renew an encryption certificate on their internal security tools, essentially overlooking a bug that would lead to a hacking breach. Even worse, Equifax did not publicize the breach until more than a month later, concealing the news away from consumers.

On the other hand, this event may ultimately also cause stakeholders of the company to lose trust in funding or providing further investment in Equifax as they have breached certain ethical code of conducts and is no longer trustworthy in the eyes of the consumers. In short, stakeholders lost faith in the company once a scandal regarding data breach occurs and was not handled properly.

Applying Guiding Principles

These problems occur as a result of lacking moral principles. All developers whether it is working on an open-source project or for a community should realize the after-effect of exploiting the source code bugs discovered. Exposing private and confidential data throughout the world may ultimately cause more problems for the society to solve, in such damaging the trust in which the society puts on the internet platform.

The developers should comply with some ethical principles such as the ACM Code of Ethics (1.2: Avoid harm) as to prevent from causing detrimental problems for the world, (1.3 Be honest and trustworthy) as to continuously develop the habit of report bugs on a transparent level, (1.6: Respect privacy) as to realize that data should only be used for legitimate purposes and not for harmful intentions. Developers acting in accordance with these ethics allow the users to feel secure when interacting with the internet world, creating a safer and healthy environment for everyone.

Generating, Evaluating and Taking Action

One of the obvious solutions to avoid the exploitation of source code is to build an extra firewall that borders the ownCloud platform. This firewall could help in monitoring the incoming data packets and only allow incoming requests whose rules defined by the administrator. It prevents unauthorized internet users from accessing the private intranet, reducing the likelihood of data breach by hackers and strengthens the security of the service.

The other viable approach is that the maintainers or users of ownCloud could hire developers to audit the open-source code to find bugs and patch them accordingly. This can be done by checking the dependencies by third parties used in the code and test them out before deploying the ownCloud platform, secure the data residing in the cloud.

Reflection

These resolutions towards the issue of source code exploitation could alleviate the risk of a data breach and reinforcing the security aspects of cloud storage when using a private open-source solution. The ACM Code of Ethics such as (1.2: Avoid harm), (1.3 Be honest and trustworthy) and (1.6: Respect privacy) would be upheld and maintained. We believe that future problem would thus be thoroughly prevented.
